General - Upgrading the Update Site version

Use the standard Eclipse plug-in upgrade mechanism to upgrade CubicTest:
  • In Eclipse select Help -> Software Updates -> Manage Configuration.
  • Locate CubicTest (expand Eclipse), right click on it and select Find Updates.... Complete the standard Eclipse plugin upgrade process.
  • When Eclipse is restarted, you need to Reset the CubicTest Perspective to get shortcuts to the new features. To reset, open the CubicTest Perspective and select Window -> Reset Perspective.
  • You should also edit your test project's pom.xml file and update the version (e.g. to 1.8.11) of the cubictest dependencies.

Upgrading from 1.8.6 to 1.8.11

Follow the procedure above. To get the new "Custom Test Suites" feature to work from maven 2, do the following.
  • Modify your project's pom.xml and add the following: 
      <repositories>
    <repository>
       <id>CubicTest</id>
       <url>http://boss.bekk.no/cubictest/m2repo/</url>
    </repository>
  </repositories>
   						
  <build>
    .....
    <sourceDirectory>src</sourceDirectory>
    <testSourceDirectory>src</testSourceDirectory>
    .....
  </build>
  
  <plugins>
     .....
     <plugin>
      	<groupId>org.apache.maven.plugins</groupId>
      	<artifactId>maven-compiler-plugin</artifactId>
      	<configuration>
      		<source>1.5</source>
      		<target>1.5</target>
      	</configuration>
     </plugin>
     .....
  </plugins>
  
  <dependencies>
    ....
    <dependency>
      <groupId>junit</groupId>
      <artifactId>junit</artifactId>
      <version>4.4</version>
    </dependency>

    <dependency>
      <groupId>org.cubictest</groupId>
      <artifactId>cubictest</artifactId>
      <version>1.8.11</version>
    </dependency>

    <dependency>
      <groupId>org.cubictest</groupId>
      <artifactId>selenium-exporter</artifactId>
      <version>1.8.11</version>
    </dependency>

    <dependency>
      <groupId>org.cubictest</groupId>
      <artifactId>cubictest-selenium-rc</artifactId>
      <version>1.8.11</version>
    </dependency>
    ....
  </dependencies>
  • Create a new CubicTest project and take a look at the CustomTestSuite.java sample file that is created. Copy the file into your existing project to get a head start at writing cusstom test suites.

General - Upgrading the Rich Client version

To upgrade the Rich Client version,
  • Download and extract the new CubicTest version as normal.
  • Copy the CubicTest / Eclipse "workspace" directory to the new version of CubicTest.
  • When CubicTest is started, you need to Reset the Perspective to get shortcuts to the new features . In CubicTest, select Window -> Reset Perspective.
