About

Welcome to web testing with CubicTest!

CubicTest is a visual tool for writing Selenium and Watir tests. It makes tests faster and easier to write, and provides abstractions to make tests more robust and reusable.

CubicTest's graphical test editor is centered around "pages" and "transitions" between pages. To be more specific, these pages are modeled as states of the application under test, and the model fits both Ajax and traditional web applications.

CubicTest is focused around writing tests that assert "what you see" on the page, and not "testing the HTML source". To support this, CubicTest has a concept called a "context" which a page element can reside in. A context can be a any HTML parent element, e.g. a part of the page, a table row, a portlet etc. Using contexts, it is easy to model user interactions such as "Press the 'Foo'-button on the row that contains the text 'Bar'" (i.e. identifying an element by specifying neighbouring elements).

Page elements can also be identified by the usual set of identifiers (e.g. "id" and "name"), and it is possible to apply numerous of them at the same time.

CubicTest comes with a Selenium RC runner (Eclipse and Maven 2 plugin), a Selenium Core exporter and a Watir exporter, but is extensible to other test frameworks as well. Writing a test-exporter is not a hard task.
